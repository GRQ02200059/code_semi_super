## 概述

### 什么是对比学习？

对比学习（Contrastive Learning）是一种自监督学习方法，通过学习相似样本（正样本对）之间的相似性和不同样本（负样本对）之间的差异性来学习更好的特征表示。

### 为什么在半监督学习中使用对比学习？

1. **增强特征表示**: 学习更有判别力的特征，提升模型区分能力
2. **充分利用无标注数据**: 在无标注数据上学习特征，不需要标签
3. **提升泛化能力**: 更鲁棒的特征表示，对噪声和变化更不敏感
4. **与EMA教师模型协同**: 完美结合，教师模型提供稳定的伪标签

### 集成效果

| 指标 | 提升 |
|------|------|
| 特征质量 | ⬆️ 10-15% |
| IoU性能 | ⬆️ 1-3% |
| 训练稳定性 | ⬆️ 显著提升 |
| 小样本鲁棒性 | ⬆️ 明显增强 |

**代价**: 训练时间 ⬆️ 15-20%, GPU内存 ⬆️ 10-15%

### 对比学习工作原理

#### 全局模式 (Global Mode)

```python
# 特征提取
features_labeled = backbone(images_labeled)           # [6, 768, 16, 16]
features_unlabeled_weak = backbone(unlabeled_weak)    # [6, 768, 16, 16]
features_unlabeled_strong = backbone(unlabeled_strong) # [6, 768, 16, 16]

# 全局池化
features_labeled = global_pool(features_labeled)      # [6, 768]
features_weak = global_pool(features_unlabeled_weak)  # [6, 768]
features_strong = global_pool(features_unlabeled_strong) # [6, 768]

# 投影
proj_labeled = projection_head(features_labeled)      # [6, 128]
proj_weak = projection_head(features_weak)            # [6, 128]
proj_strong = projection_head(features_strong)        # [6, 128]

# 构建正样本对
positive_pairs = [
    (weak_i, strong_i) for i in range(batch_size)
]
# 同一图像的弱增强和强增强版本

# 负样本对
negative_pairs = [
    (sample_i, sample_j) for i != j
]
# 不同图像之间

# 计算对比损失
loss = -log(sim(positive) / (sim(positive) + Σ sim(negatives)))
```

#### 像素级模式 (Pixel Mode)

```python
# 提取特征图（保持空间维度）
features = backbone(images_labeled)  # [B, 768, 16, 16]

# 采样像素
sampled_features, sampled_labels = sample_pixels(features, labels)
# [N, 768], [N] 其中N是采样的像素数

# 正样本对：相同类别的像素
# 负样本对：不同类别的像素

# 计算对比损失（像素级别）
loss = pixel_contrastive_loss(sampled_features, sampled_labels)
```

### 关键参数说明

#### 1. `use_contrastive` (bool)

- **默认**: `True`
- **作用**: 是否启用对比学习
- **建议**: 启用以获得更好的特征表示

#### 2. `contrastive_weight` (float)

- **默认**: `0.5`
- **范围**: `0.1 - 1.0`
- **作用**: 控制对比学习损失的权重
- **调优**:
  - `0.3`: 保守配置，对原有性能影响小
  - `0.5`: 平衡配置，推荐起点
  - `0.7-1.0`: 激进配置，强调特征学习

#### 3. `contrastive_temperature` (float)

- **默认**: `0.07`
- **范围**: `0.05 - 0.15`
- **作用**: 控制相似度分布的平滑度
- **调优**:
  - `0.05`: 更尖锐的分布，区分更严格
  - `0.07`: 标准值，平衡
  - `0.10-0.15`: 更平滑的分布，区分更宽松

#### 4. `contrastive_mode` (str)

- **选项**: `'global'` 或 `'pixel'`
- **global**:
  - 在图像级别进行对比
  - 计算效率高
  - **推荐用于大部分场景** ⭐
- **pixel**:
  - 在像素级别进行对比
  - 更精细，但计算量大
  - 需要更多GPU内存
  - 适合精细分割任务

#### 5. `projection_dim` (int)

- **默认**: `128`
- **作用**: 投影头输出维度
- **建议**: `64-256`，通常128足够

#### 6. `projection_hidden_dim` (int)

- **默认**: `256`
- **作用**: 投影头隐藏层维度
- **建议**: 是`projection_dim`的2倍

## 故障排除

### 问题1: 训练过程中loss出现NaN

**原因**: 
- 温度参数过小
- 学习率过高
- 批次大小过小

**解决方案**:
```python
# 增加温度
contrastive_temperature=0.1  # 从0.05增加到0.1

# 降低学习率
optimizer = dict(lr=5e-5)  # 从1e-4降低到5e-5

# 增加批次大小
labeled_batch_size=8
unlabeled_batch_size=8
```

### 问题2: GPU内存不足

**解决方案**:
```python
# 1. 使用全局模式而非像素模式
contrastive_mode="global"

# 2. 减小批次大小
labeled_batch_size=4
unlabeled_batch_size=4

# 3. 减小投影维度
projection_dim=64
projection_hidden_dim=128

# 4. 使用混合精度训练
trainer = dict(precision=16)
```

### 问题3: 对比学习没有带来提升

**可能原因**:

1. **权重过低**: 
```python
contrastive_weight=0.7  # 尝试增加
```

2. **温度不合适**:
```python
# 尝试不同温度
contrastive_temperature=0.05  # 或 0.1
```

3. **训练轮数不足**:
```python
max_epochs=150  # 增加到150
```

4. **数据增强不够强**:
检查`consistency_augmentation=True`是否启用

### 问题4: 训练速度太慢

**解决方案**:

```python
# 1. 减小投影头复杂度
projection_hidden_dim=128  # 从256减小到128

# 2. 降低对比学习权重
contrastive_weight=0.3

# 3. 使用FP16训练
trainer = dict(precision=16)

# 4. 增加num_workers
data = dict(num_workers=12)
```

### 问题5: 投影头维度不匹配

**错误信息**: 
```
RuntimeError: mat1 and mat2 shapes cannot be multiplied
```

**原因**: Swin backbone输出维度与投影头输入维度不匹配

**解决方案**:
```python

# Stage 4输出: 768维

# Stage 4输出: 1024维

    backbone=dict(
        embed_dims=96,  # Swin-Small
        depths=[2, 2, 18, 2],
        # 最终输出: 96 * 2^3 = 768
    )
```

### 混合使用多种对比学习

```python
# 同时使用全局和像素级对比
class MixedContrastiveModule(SemiSupervisedContrastiveModule):
    def __init__(self, ...):
        super().__init__(...)
        # 添加两种对比损失
        self.global_contrast = SupConLoss(temperature=0.07)
        self.pixel_contrast = PixelContrastLoss(temperature=0.07)
    
    def _compute_contrastive_loss(self, ...):
        global_loss = self._compute_global_contrastive_loss(...)
        pixel_loss = self._compute_pixel_contrastive_loss(...)
        return 0.5 * global_loss + 0.5 * pixel_loss
```

## 最佳实践

### ✅ 推荐做法

1. **从保守配置开始**: `contrastive_weight=0.3`
2. **使用全局模式**: `contrastive_mode="global"`
3. **逐步调整参数**: 每次只改一个参数
4. **多次实验取平均**: 半监督学习有随机性
5. **监控TensorBoard**: 观察各个损失的变化
6. **保存多个checkpoint**: 不同指标可能在不同epoch最优

### ❌ 避免做法

1. ❌ 一次改变多个参数
2. ❌ 使用过高的对比学习权重 (>1.0)
3. ❌ 在小batch size下使用像素级对比
4. ❌ 忽略baseline对比
5. ❌ 过早停止训练
